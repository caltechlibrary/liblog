Reusing Assets & Shared Content

Reuse as much content as possible, including pages, boxes, links, databases, books, and widgets, to be consistent and reduce maintenance.

What are assets?

Assets are reusable links in LibGuides that are maintained in one place.  Use assets for databases (e.g., Academic Search Premier) and common online resources (e.g., Purdue OWL) instead of creating new links.

Why use them?  When you use an asset, you reduce the likelihood of inconsistent and broken links.  Assets are updated once from the master asset list, which automatically updates the assets used throughout our guides.  Assets can only be used in Link and Database boxes, not Rich Text/HTML boxes.

Database assets
  1. In your box, click Add / Reorder > Database.
  2. Search for and select the database.
  3. Decide whether to use the default description or write a custom description.
  4. Select description location: under the title, on title hover, or an i icon. Click Save.
Link assets
  1. In your box, click Add / Reorder > Link.
  2. Click the tab, Reuse Existing Link.
  3. Search for and select the asset you would like to use. You can change the description after adding the link if you'd like.  Click Save.
Book assets
  1. In your box, click Add / Reorder > Book from the Catalog.
  2. Click the tab, Reuse Existing Book.
  3. Search for the title of the book and select one from the drop-down. If one doesn't appear, then there is no asset for that book yet.

Where can I find reusable content?

Everything in LibGuides - guides, pages, boxes, books from the catalog, links, widgets, etc. - is reusable.

  • When adding a new item, look for tabs or buttons such as Reuse Existing Link/Box/Widget, or Copy content/layout.
  • Use the Library Component Storage Guide to find reusable widgets and boxes.

Media

Images

Ensure that your images are useful, responsive, and accessible.

Technical advice
  • Create your own images if possible and scale/resize them before you upload them.
    • How to resize:
    • Don't fix the height/width so images remain responsive to the window size.
    • Compression:
  • Avoid using screenshots as they quickly become outdated.
  • Upload images to the LibGuides Image Server rather than copy/paste to prevent broken images.
  • The best file formats for web-safe images are .gif, .png, and .jpg.
  • Use descriptive file names (not asd98yfbj.jpg).
Accessibility
  • Add alternative text to make images accessible (e.g., screen reader-friendly) and discoverable.
  • Include a caption for images that feature text so users with screen-readers can receive the same content.
Attribution

Use free, legal images, and add attribution to images that require it.  Attributions should include the title, author, source, and license.

  • Title – the name of the material (if available)
  • Author – the owner of the material (this could be a person's name, a company name, or a user name on a site such as Flickr)
  • Source – where the image can be found (usually a URL or hyperlink)
  • License – which Creative Commons license the image has (e.g., CC BY 2.0 )

Example of a complete citation for a Creative Commons image: " Creative Commons 10th Birthday Celebration San Francisco " by tvol is licensed under CC BY 2.0

Open Washington provides a handy Open Attribution Builder tool that makes it easy to generate citations.

Video

  • Videos must include transcripts or captions in order to be accessible.
