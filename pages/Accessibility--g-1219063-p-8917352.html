Accessibility Requirements

This list does not present all accessibility issues, but by addressing these basic principles, you will ensure greater accessibility of your web content to everyone.

  • Check the accessibility of your page:
    • WAVE Accessibility Tool (also available as a handy Firefox/Chrome extension )
  • Helpful resource:
    • Princeton University Digital Accessibility

Content

  1. Provide descriptive, unique page titles.
  2. Organize content with properly nested headings.
  3. Write meaningful text links.
  4. Captions must be provided for all videos.
  5. Provide clear instructions that don't rely solely on sensory cues.
  6. Write concisely in simple, plain language.

Color and contrast

  1. Never use color alone to visually convey important information.
  2. Ensure colors on the page and in images, graphs and charts have adequate contrast.

Images & Video

  1. Supply equivalent alternate text for images in the content itself, a caption or an ALT attribute.
  2. The ALT attribute of linked images should describe both the content of the image and the function of the link.
  3. Provide a blank ALT attribute on decorative images.
  4. Images that contain text should be avoided.
  5. Videos should be captioned for posting.

Links

  1. Write meaningful text links. Adjust links to make sure linked text is a string rather than just a single word (e.g., ‘here’) or a URL.
  2. When linking to documents, indicate the document title and type of file.
  3. The ALT attribute on linked images should describe both the content of the image and the function of the link.
  4. Don't rely solely on color to convey links.
  5. Linked documents (such as PDF and Word docs) should be accessible.
  6. Fix links (e.g., broken links or links that no longer make sense).
  7. Most links should open in same tab by default. Reasoning:
    • For users who rely on back arrow navigation, it’s confusing when they aren’t able to use it to get back to the previous page.
    • There are options to force a link to open in a new tab/window (e.g. right-click, select) but you can’t easily “unforce” a link to open in a new tab/window.
    • Mobile considerations: reduces the chance of mobile users getting lost or unable to locate a previous tab. Opening in the same tab also keeps the “back button” as an option for users to return to the previous page, which they wouldn’t be able to utilize if they were in a new tab and maybe didn’t know it.
    • Best for navigation for screen readers/accessibility.
    • Gives users choice to right-click and open in new tab (there is no similar choice to open in same tab)
    • Consistency with web conventions is important for usability.
    • There is a security concern related to opening links in a new tab. A vulnerability of target=”_blank” attribute may leave users open to a phishing attack unless the target=”_blank”attribute is accompanied by rel="noopener" attribute.

Navigation

  1. The layout of elements on a website should be consistent on every page.
  2. Use properly nested headings to structure your document.
  3. Write meaningful link text.

Tables

  1. Tables are used for data, not layout. If used for data, tables are more likely to be accessible.
  2. Captions should be included where possible.
  3. Don't define a width for tables or table cells.
